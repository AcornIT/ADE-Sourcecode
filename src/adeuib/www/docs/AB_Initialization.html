1. Overview

AppBuilder is a GUI tool for developing Progress GUI and character applications. This document describes AppBuilder’s Initialization Process only. This process begins when AppBuilder is first invoked (for example, when the user presses the AppBuilder button on the ADE Desktop or double-clicks an AppBuilder-generated source file). The AppBuilder Initialization Process ends when AppBuilder has been fully realized and is waiting for user input.

Note: There are many references to ‘UIB’ throughout the AppBuilder source code. When AppBuilder was first developed, it was commonly referred to as the UIB (User Interface Builder). Although the product has developed into and been renamed AppBuilder, many of the source files and routines still contain references to ‘UIB’ and sometimes contain ‘UIB’ in the source file name, for example _uibstrt.p.

2 Architecture

AppBuilder has many functions it must perform during the initialization process:

  • Defining standard default colors and fonts
  • Prompting for logins if necessaryChecking to see if the operating system supports AppBuilder
  • Starting up a library of persistent procedures that can be called anywhere in AppBuilder
  • Defining many internal procedures that will be used during the execution of AppBuilder
  • License-checking (AppBuilder, WebSpeed Workshop etc…)
  • Laying out, Creating and Realizing the appropriate window/menus/toolbars
  • Generating the Most-Recently Used List (MRU)

4. Primary Algorithms and Processes

AppBuilder can be started in many different ways. Here are a few examples:

  • Pressing the AppBuilder button from the Desktop
  • Double-clicking on an AppBuilder-generated source file (.w)
  • Running it from the command line or from an MS-Windows Icon:

prowin32 –p _ab.p

The following diagram illustrates the logical flow of the main initialization routines.

The main entry point for AppBuilder is src/_ab.p. This program does the following:

  • Loads standard default colors and fonts (adecomm/_adeload.p)
  • Reads the default environment from the .ini file or registry
  • Sets the MS-Windows ALT-TAB icon
  • Handles logins if necessary (src/_prostar.p)
  • Runs adeuib/_uibmain.p passing it a window list if there is one

Adeuib/_uibmain.p is the main routine for AppBuilder. It defines several temp-tables as well as many internal procedures that AppBuilder calls during regular processing (not just initialization). It calls internal and external procedures to do the following:

  • Starts up a function library persistently for use all over AppBuilder (adeuib/_abfuncs.p)
  • Creates a widget pool
  • Defines UIB Internal procedures (adeuib/uibmproa.i, adeuib/uibmproe.i, adeuib/uibmundo.i)
  • After the initial definitions, runs adeuib/_uibstrt.p which calls internal and external procedures to do the following:
    • Check for a valid license (adeshar/_ablic.p)
    • Establish a db cache
    • Create a window for the bar menu and icon palette
    • Define the Advanced/Color editor support
  • Based on the licensing (AppBuilder or WebSpeed Workshop), creates the AppBuilder window objects so that they have all of the proper icons, buttons, and functionality
  • Creates COM objects to support the ActiveX controls palette
  • Handles customizing
  • Starts up the AppBuilder window in its initial form (either a new window open, or existing window open, or nothing special)
  • Initializes the library manager and window manager
  • Defines trigger code for the AppBuilder window

8. Process and Requirements

The following tips will help you build your changes into the Progress ADE Tools.

  • Set up your directory structure so that your working directory is at the src level.
  • Have subdirectories such as adeuib and adecomm underneath your working directory. Make your changes into these directories.
  • Run Progress from your working directory.
  • Run Progress using the –zn startup parameter so that you will be able to compile source code files that begin with _ (underscore).
  • Be sure that the location of your full ADE source is placed at the end of your PROPATH so that you will be able to pick up include files that you have not moved to your working directory structure.

See the training presentation, How to Be a POSSE Programmer, for examples of how to build your changes into the Progress ADE tools.

11. Supplementary Information

You might find it useful to compile adeuib/_uibmain.p using the PREPROCESSOR flag on the COMPILE statement when you are first looking at this code. It will create one large source file that has all the include files inserted and all the text substitutions performed. This might make it easier to familiarize yourself with the control flow before you actually begin changing components of the source code.
