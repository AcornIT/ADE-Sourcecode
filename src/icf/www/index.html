Internet Component Framework (ICF)

The Internet Component Framework (ICF) is an application development framework that speeds the assembly and deployment of business applications for Internet-based and traditional computing models (e.g., Internet B2B, Internet B2C, ASP, client/server, host-based, etc.). The ICF features components that handle application services, such as context management, security, and customization, to minimize the amount of code developers need to build for any application. The ICF embodies the prescriptions and best practices of building applications using the Progress 4GL, such as Progress' Universal Application Architecture (UAA) and Application Development Model (ADM).

 

ICF Version 1.1 Available for Download!

ICF Version 1.1 is now available for download below. The primary purpose of Version 1.1 is to provide those features and functions most critical for developers to begin using the framework to build real world, deployable applications. Version 1.1 has been submitted for your review and comment. It has not been fully tested or accepted by the Core Review Team for merger into the new features branch. Caution should be exercised in the manner in which you use this submission. A list of known issues in ICF Version 1.1 is available.

Note that ICF Version 1.1 makes use of several ActiveX controls. The Baseline includes an installation program, called ICFCtrlSetup.exe. You will need to run this program to install the correct controls on your system.

The latest ICF code is also available for download via CVS. ICF Development with WinCVS describes this download process.

Initial ICF Code Submission

The initial submission (Version 0.9, posted for download in June of 2001) has been pulled down, as it will be replaced in the very near future and should not be used for new development. The ICF Submission 0.9 Archive could only deploy the repository via Roundtable; the purpose of Version 1.0 is to add the ability to deploy the repository via CVS, the POSSE code version control system. Version 1.0 also incorporates some number of bug fixes since Version 0.9.

 

ICF Specifications Available for Review

From time to time, new ICF specifications will be made available as the project evolves. The User Interface Enhancements Design Specification (19-Jul-2001) provides a design specification for a revised AppBuilder toolbar and User Interface that integrates the ICF Development Menu and its tools into the AppBuilder. ERwin Integration Functional Specification (18-Jul-2001) describes the use and functionality of the ICF ERwin template and associated macrocode. Details all ICF functionality relating to ERwin integration and general database design through ERwin. Check out the ICF section of the Projects Documetation page for the latest listing.

Foundation Framework Chosen

Mip Holdings' Astra framework was selected for its comprehensive repository, security, session and data management components. These components follow the guidelines of Progress' Universal Application Architecture (UAA), providing a framework for rapidly building applications that can communicate easily, support multiple user interfaces (Web, GUI, CHUI), and evolve with technology without the need to redesign or rewrite from scratch.
